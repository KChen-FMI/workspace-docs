# FMI Azure Tenant Migration Plan

作成日: 2026-02-13
対象: Azure VMのうち `OSサポート期限 = 2027-01-12` のサーバ（ただし X-point はSaaS移行のため除外）

## 1. 目的
- 現在の Azure 環境（E5と別テナント）上の対象VMを、OS更改と同時に E5 と同一テナントへ移行する。
- 2027年1月サポート期限のリスクを解消し、運用を E5 テナントへ統合する。

## 2. スコープ
### 2.1 対象（インフラ移行）
- FMISV304: AD#1 (FSMO/ADCS/DNS)
- FMISV302: D365 バッチサーバ
- FMISV224 (Azure上: FMISV307): SKYSEA
- FMISV309: SKYSEA HTTP-GW

### 2.2 非対象（別トラック）
- FMISV306: X-point2.7
  - 理由: X-point は SaaS へ移行するため、IaaS VM移行計画から除外

## 3. 基本方針
- 方式: Side-by-Side（新規構築して切替）
- AD/ADCS/DNS を最優先で先行移行し、認証・名前解決の土台を先に固める。
- 旧テナントと E5 テナント間は移行期間のみ接続し、移行完了後に分離。
- 本番切替は業務影響を最小化するため段階実施。

## 4. 依存関係と推奨順序
1. 事前準備（Landing Zone / 接続 / 運用基盤）
2. AD基盤移行（FSMO, DNS, ADCS）
3. AAD Connect 切替（必要に応じて）
4. SKYSEA本体移行（FMISV224/307）
5. SKYSEA HTTP-GW移行（FMISV309）
6. D365バッチ移行（FMISV302）
7. 安定化・旧環境廃止

## 5. フェーズ計画
### Phase 0: 事前準備
- E5テナントに移行先サブスクリプション、VNet/Subnet、NSG、監視、バックアップ、ログ基盤を構築
- 名前解決、時刻同期、ルーティング、FWルールを設計
- 旧テナントとの一時接続を設計（疎通・セキュリティ要件を明文化）
- 非機能要件（RTO/RPO、監査、運用時間帯）を確定

### Phase 1: AD移行（最優先）
- 新DCをE5テナント側で構築（推奨: Windows Server 2022）
- 既存ドメインへ参加、レプリケーション確認
- FSMO移管、DNS切替、サイト/サブネット定義更新
- ADCS移行（CA証明書、テンプレート、CRL配布含む）
- 旧ADサーバ（FMISV304相当）は段階的に降格・退役

### Phase 2: AAD Connect/ID連携
- 現行同期構成を棚卸
- 必要に応じて Staging 構成で新サーバ準備、切替リハーサル
- 本番切替後、同期エラーと重複属性を解消

### Phase 3: 管理系サーバ移行（SKYSEA）
- FMISV224/307 を E5 テナントに新規構築し、設定・データを移行
- FMISV309（HTTP-GW）を移行し、外部接続経路を切替
- 監視・証明書・公開設定を新環境へ統合

### Phase 4: 業務系サーバ移行（D365bat）
- FMISV302 を新環境へ構築し、バッチジョブ移行
- 並行稼働期間を設けて実行結果比較
- 切替判定後、本番ジョブを新環境へ一本化

### Phase 5: 安定化と廃止
- 監視/バックアップ/運用手順をE5テナントへ統一
- 旧テナント側の対象資産を停止・削除（監査ログ保持方針に従う）
- 事後レビュー（課題・再発防止）を実施

## 6. マイルストーン（目安）
- M1: 設計完了（2026-03）
- M2: AD移行完了（2026-06）
- M3: SKYSEA系移行完了（2026-08）
- M4: D365bat移行完了（2026-10）
- M5: 旧環境廃止完了（2026-11）

注: サポート期限 2027-01-12 に対し、遅くとも 2026-10 までに本番切替完了を目標とする。

## 7. リスクと対策
- ADCS移行不備による認証障害
  - 対策: 事前に証明書チェーン/CRL配布先を検証し、切替手順をリハーサル
- DNS切替ミスによるアプリ疎通不良
  - 対策: TTL短縮、段階切替、ロールバック手順の明文化
- SKYSEA/HTTP-GW切替時の外部接続影響
  - 対策: 公開経路の並行稼働期間を設定し監視強化
- D365バッチ結果差異
  - 対策: 並行実行で結果比較し、差分解消後に切替

## 8. 直近アクション
1. FMISV302 の最終スコープ（移行対象）を確定
2. AD/ADCS 詳細設計ワークショップを実施
3. E5テナント側Landing Zone構築タスクを起票
4. SKYSEAベンダーと移行支援範囲を確定
5. X-point SaaS移行計画を別紙化（本計画の依存として管理）
