# FMI ITサービスカタログ（社内向け）

最終更新日: 2026-02-19  
対象: FMI グループ従業員（正社員・契約社員・派遣社員）

## 1. 位置づけ（ServiceNow定義に基づく）

本カタログは、**社内で利用可能な IT リソースを一元化したデータベース**であり、利用者ニーズに対して効率的・効果的に応えるための標準窓口です。  
また、カタログ（何が頼めるかの定義）とポータル（申請・進捗確認の UI）を分離し、セルフサービス性を高めます。

## 2. カタログ設計原則

- 利用者にとって分かる名称で記載する
- サービスごとに `カテゴリ / 説明 / オーナー / 可用性 / SLA / コスト` を明示する
- 関係者が不要な情報や機密情報は掲載しない
- 利用ログを継続的に見直し、四半期ごとに改善する

## 3. 対象範囲

- 対象: 社内 IT サービス要求（端末、アカウント、アプリ、権限、サポート等）
- 非対象: セキュリティ詳細設定値、管理者向け運用手順、監査用機微情報

## 5. 申請時に必須の入力項目

- 申請者情報（氏名、所属、連絡先）
- 対象ユーザー（本人/代理）
- 希望サービスID
- 希望対応期限と業務影響
- 承認者情報
- 添付情報（見積、エラー画面、対象ファイル等）

## 6. 優先度ルール（共通）

| 優先度 | 定義 | 目標一次応答 |
|---|---|---|
| P1 | 全社業務停止・重大インシデント | 30分以内 |
| P2 | 部門業務に重大影響 | 2時間以内 |
| P3 | 個別ユーザー影響（回避策あり） | 1営業日以内 |
| P4 | 問い合わせ・軽微依頼 | 2営業日以内 |

## 7. 掲載しない情報（運用方針）

- 脆弱性情報、管理者資格情報、ネットワーク詳細構成
- 監査証跡の生データ、個人情報を含む運用ログ
- 利用者に不要な内部実装手順

## 8. 運用KPI（四半期レビュー）

- サービス別申請件数
- SLA遵守率
- 初回解決率（FCR）
- 再申請率（申請フォーム不備含む）
- 利用者満足度（CSAT）

## 9. 改訂ガバナンス

- オーナー: IT部門（サービスデスク責任者）
- レビュー頻度: 四半期ごと
- 改訂トリガー: 新システム導入、規程改定、重大障害の再発防止
